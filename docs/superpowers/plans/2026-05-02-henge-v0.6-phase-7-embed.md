# Henge v0.6 — Phase 7: Embeddings refactor

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development.

**Goal:**
1. Default OpenAI embedding model: `text-embedding-3-small` → `text-embedding-3-large` (~15-25% better Spanish recall, cost despreciable).
2. Add `EMBED_AVERAGE` flag (default `false`). When true, run BOTH OpenAI + Voyage embeddings, average their cosine-distance matrices, then project. Reduces single-model bias.

**Architecture:**
- Modify `henge/embed.py`:
  - Change default model.
  - When `EMBED_AVERAGE=true`, call both `_embed_openai` and `_embed_voyage`, return dict with both embedding sets.
  - `project_mds` accepts either flat list (single provider) or list-of-2-sets (averaged) and handles both.
- Default `false` because averaging requires both `OPENAI_API_KEY` and `VOYAGE_API_KEY`. Graceful fallback when one provider missing.

**Out of scope:**
- Refactoring to a formal `EmbeddingProvider` ABC (overkill for 2 providers; functional approach matches existing style).
- New CLI flag — env var only.
