# Henge v0.6 — Phase 5: Tenth-man dual (blind + informed)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Pasar de un único tenth-man (Opus, ve los 9) a un par dual:
- **Blind** (Opus 4.7) — ciego, no ve los 9, produce el dissent más puro posible. Métrica de distancia honesta usa este.
- **Informed** (gpt-5) — ve los 9 + el blind, reconcilia: qué se sostiene del blind cuando ve los 9, qué se revisa, qué se descarta. Cross-lab second opinion sobre el blind de Opus.

**Architecture:**
- Nuevo módulo `henge/tenth_man.py` con dataclasses + 2 funciones públicas (`run_tenth_man_blind`, `run_tenth_man_informed`).
- `henge/agents.py::run_agents` orquesta: paralelo(9 frames + blind), luego informed secuencial. Drop del `client` parameter (Phase 5 finaliza la migración legacy).
- `henge/server.py` deja de instanciar `AsyncAnthropic()` localmente.
- `henge/viz.py` renderiza dos secciones: tenth-man blind (métrica) + tenth-man informed (refinamiento).
- Métrica `distance_to_centroid_of_9` para el reporte sigue usando el embedding del BLIND, no del informed.
- Feature flag `HENGE_ENABLE_TENTH_MAN_DUAL` (default `true`). Si false, fallback al modo v0.5 (un solo Opus).

**Cross-lab rationale:** blind crea el dissent puro (Opus solo, sin contaminación). Informed lo audita desde otra escuela (gpt-5 con todo el contexto). Si gpt-5 mantiene grandes pedazos del blind, dissent robusto. Si gpt-5 los descarta, eran sesgo del lab.

### Task 5.2: `run_tenth_man_blind` (Opus, ciego)

Reuse `henge/prompts/10-tenth-man-steelman.md`. Build user message that says "no consensus has been generated yet — anticipate any plausible consensus on this question and produce the strongest dissent". User message: question + canonical_context only.

Use `complete("anthropic/opus-4-7", req)`. `max_tokens=6000` (TENTH_MAX_TOKENS already there). Opus is reasoning model — DON'T set `reasoning_effort` (Anthropic provider ignores it; Opus already optimized).

### Task 5.3: `run_tenth_man_informed` (gpt-5, ve los 9 + blind)

New prompt template (inline string literal in `tenth_man.py`, not a separate `.md` file — keeps cohesion). Asks gpt-5:
- Read the 9 frames + the blind dissent
- Output structured JSON: `{ text, what_holds, what_revised, what_discarded }`

Use `complete("openai/gpt-5", req)` with `reasoning_effort="low"`, `max_tokens=4000`. JSON tolerant parsing.

### Task 5.4: Refactor `run_agents` + drop `client` param

New signature: `run_agents(question, context=None, temperature=TEMPERATURE)`. No client.

Body:
```python
nine_task = asyncio.gather(*[run_agent(...) for f in FRAMES], return_exceptions=True)
blind_task = run_tenth_man_blind(question, context)
nine_raw, blind_result = await asyncio.gather(nine_task, blind_task)
# build nine list, abort if <8 ok
informed_result = await run_tenth_man_informed(question, context, successful_nine, blind_result)
return {"nine": nine, "blind": blind_result, "informed": informed_result}
```

Return shape changes from `list[(frame, text, status, usage)]` length 10 to a dict with three keys. Server.py adapts.

Keep legacy `run_tenth_man(client, ...)` and the legacy `run_agents(client, ...)` signature as deprecation shims that delegate to the new path. Phase 8 removes the shims.
